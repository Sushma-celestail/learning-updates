CSS Specificity Challenge

10 list items styled with progressively more specific selectors — no !important, no inline styles.

  • Item 1 — first-child (font-size 20px, uppercase, odd bg)
  • Item 2 — .highlight (color blue, even bg)
  • Item 3 — plain odd item
  • Item 4 — .highlight (color blue, even bg)
  • Item 5 — #special + .highlight (red wins: 1-0-1 > 0-1-1)
  • Item 6 — plain even item
  • Item 7 — .highlight (color blue, odd bg)
  • Item 8 — plain even item
  • Item 9 — .highlight (color blue, odd bg)
  • Item 10 — last-child (italic)
Specificity reference — all rules in this file
li 0 – 0 – 1 Base: 16px gray. Lowest score — foundation for everything.
li:nth-child(even) 0 – 1 – 1 Light yellow background on even items.
li:nth-child(odd) 0 – 1 – 1 Light gray background on odd items.
li:first-child 0 – 1 – 1 20px + uppercase. Same score as :nth-child but declared later → wins font-size via cascade.
li:last-child 0 – 1 – 1 Italic on item 10.
li:hover 0 – 1 – 1 Underline + scale(1.02) on mouse-over.
li.highlight 0 – 1 – 1 Blue color. Wins over :nth-child color via source order.
li#special 1 – 0 – 1 Red + bold. ID column makes this the highest-specificity rule — overrides .highlight's blue even though both match item 5.
