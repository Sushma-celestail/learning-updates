position: absolute (tooltip)

Hover Tooltips

The .tooltip-wrapper is position: relative. The tooltip itself is position: absolute, offset above the icon. It transitions from invisible to visible on :hover — pure CSS, no JavaScript.

/* Parent — tooltip is positioned relative to this */ .tooltip-wrapper { position: relative; } /* Tooltip — hidden by default */ .tooltip { position: absolute; bottom: calc(100% + 12px); opacity: 0; visibility: hidden; transition: opacity .22s, visibility .22s; } /* Shown on hover */ .tooltip-wrapper:hover .tooltip { opacity: 1; visibility: visible; }

position: static

Normal Page Flow

These blocks use the browser default — position: static. They stack vertically in document order and cannot be offset with top, left, etc. This is 99% of the page.

/* Static is the default — no positioning context */ .content-block { position: static; /* default — flows normally */ /* top / left / right / bottom have NO effect here */ }
🧭

Normal Flow

Static elements participate in normal document flow. They push siblings down and don't create a stacking context.

📐

No Offset

You cannot move a static element with top, right, bottom, or left. Those properties are ignored.

🔢

No z-index

z-index has no effect on static elements either. Use relative, absolute, fixed, or sticky to control layering.

🌊

The Default State

Every element starts as static. You only add other positioning values when you need to break out of the normal flow for a reason.

📦

Block-level boxes

Block elements like div, p, section stack vertically. Inline elements like span sit on the same line.

🔍

Readable Source Order

Keeping content in static flow maintains accessible source order — screen readers and tab focus follow document order naturally.

📌

Sticky vs Fixed

Sticky elements start in the document flow and "stick" only after scrolling past their original position. Fixed elements are always removed from flow and attached to the viewport.

🪄

z-index Stacking

The nav has z-index: 100, tooltips have z-index: 50, and the fixed button has z-index: 200 — preventing any content from overlapping them incorrectly.

🎯

Pure CSS

Every positioning effect on this page — the sticky nav, absolute badges, hovering tooltips, and fixed button — uses only CSS. Zero JavaScript.
